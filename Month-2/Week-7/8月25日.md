## 响应式和自适应

### 区别

"响应式"（Responsive）和"自适应"（Adaptive）是两个用来描述网页和应用程序适应不同屏幕尺寸和设备的术语。虽然它们通常用来表达相似的概念，但它们在细微的方面有一些区别。

**响应式设计（Responsive Design）**：

响应式设计是一种以适应不同屏幕尺寸和设备为目标的设计方法。在响应式设计中，网页的布局和内容会根据浏览器窗口的大小和设备的屏幕尺寸进行调整。主要特点包括：

- 使用 CSS 媒体查询（Media Queries）来根据不同的屏幕宽度应用不同的样式。
- 元素的大小、位置、布局等会根据不同的视口尺寸进行动态调整，以适应不同的设备。
- 响应式设计通常基于流式布局，元素的相对宽度（百分比）更常见，以适应不同屏幕。
- 网页会在不同的屏幕上重新排列内容，以提供更好的用户体验。

**自适应设计（Adaptive Design）**：

自适应设计是一种根据特定屏幕尺寸和设备类型进行适应的设计方法。在自适应设计中，网页会根据预定义的断点（breakpoint）调整样式和布局。主要特点包括：

- 预定义多个断点，每个断点对应一组样式规则，用于特定屏幕尺寸和设备类型。
- 在不同的断点上，网页可能会重新排列内容、隐藏某些元素、更改字体大小等，以适应不同的屏幕。
- 自适应设计更加精确地控制在特定屏幕尺寸下的布局和样式。

**区别**：

- 响应式设计是更灵活的，适应各种屏幕尺寸和设备。自适应设计则更专注于特定的断点和设备类型。
- 响应式设计使用媒体查询来动态调整样式。自适应设计则在预定义的断点上使用不同的样式规则。
- 响应式设计的布局更流式，元素的大小和位置更自由。自适应设计的布局更加精确，根据断点调整。
- 响应式设计通常需要使用相对单位来适应不同屏幕。自适应设计可以使用固定单位，因为样式在特定断点上固定。

总的来说，响应式设计更加适合应对不同屏幕和设备，而自适应设计更加精确地适应特定的屏幕尺寸和设备类型。在实际开发中，可以根据项目的需求和目标选择适合的设计方法。

### 联系

响应式设计和自适应设计都是为了确保网页和应用程序能够在不同屏幕尺寸和设备上呈现出最佳的用户体验。尽管它们在某些方面有区别，但它们之间也有一些联系和交叉点。

**联系点**：

1. **适应不同设备**：无论是响应式设计还是自适应设计，它们的共同目标都是使网页在多种设备上都能正常运行，包括桌面电脑、平板电脑和移动设备等。

2. **CSS 媒体查询**：响应式和自适应设计都使用 CSS 媒体查询来检测屏幕特性，如宽度和高度，从而应用不同的样式规则和布局。

3. **断点**：虽然在自适应设计中更加明确，但响应式设计也可以通过断点来调整布局和样式，以适应不同屏幕尺寸。

4. **流式布局**：响应式设计和自适应设计都倾向于使用流式布局，以便元素能够自适应屏幕尺寸。

5. **用户体验优化**：无论是响应式还是自适应，它们的共同目标之一都是提供出色的用户体验，使用户能够轻松浏览和与内容交互。

**交叉点**：

1. **结合使用**：实际开发中，响应式和自适应的设计方法可以结合使用。例如，在响应式设计的基础上，使用自适应设计来针对特定设备类型进行微调。

2. **浏览器兼容性**：两种方法都需要考虑不同浏览器和设备的兼容性，以确保页面在各种环境中正常显示。

3. **移动优先设计**：现代设计趋势中，常常会采用移动优先设计策略，即首先设计适合移动设备的布局和样式，然后再逐步扩展到更大的屏幕尺寸。这既有响应式又有自适应的元素。

总的来说，响应式设计和自适应设计在实际应用中可以相互补充和交叉使用，以满足不同项目的需求和目标。无论你选择哪种方法，都应考虑到用户体验、页面性能和不同设备的兼容性。

## 流式布局

流式布局（Fluid Layout）是一种网页布局设计方法，旨在使网页内容能够根据不同屏幕尺寸和设备动态地自适应调整。流式布局的主要目标是确保网页内容在不同屏幕上都能正常显示，无论是在大屏幕的桌面电脑上，还是在小屏幕的移动设备上。

流式布局的特点包括：

1. **相对单位**：在流式布局中，相对单位如百分比和 em/rem 更常见，而不是绝对单位如像素（px）。使用相对单位可以使元素的大小和位置相对于其容器或视口大小进行调整。

2. **自动缩放**：元素的宽度和高度不是固定的像素值，而是基于父元素的宽度或视口的大小来计算。这使得元素能够自动缩放以适应不同的屏幕。

3. **弹性布局**：元素的布局更加弹性，可以根据可用空间的大小自动调整。这意味着元素之间的间隔和位置可能会随着屏幕尺寸的变化而改变。

4. **媒体查询**：流式布局通常结合使用 CSS 媒体查询，以根据不同的屏幕宽度应用不同的样式。媒体查询可以在特定的断点上调整布局和样式。

优点：

- 提供了更好的跨设备兼容性，确保网页在不同屏幕尺寸上的良好呈现。
- 适应用户使用各种设备浏览网页的需求，提高了用户体验。
- 适用于响应式设计，使网页适应从桌面电脑到移动设备的所有尺寸。

缺点：

- 在某些情况下，可能需要额外的样式调整，以确保元素在不同尺寸上显示正确。
- 可能需要更多的测试，以确保在各种设备和浏览器中都能正常工作。
- 对于某些特定布局要求，可能需要额外的 CSS 或 JavaScript 来实现更复杂的效果。

总的来说，流式布局是一种灵活的设计方法，可以确保网页内容在不同屏幕上的适应性。结合使用媒体查询和相对单位，可以实现更好的跨设备体验。

## em和rem

`em` 和 `rem` 都是用于设置字体大小和元素尺寸的 CSS 相对单位。它们与绝对单位（如像素）不同，可以根据父元素或根元素的大小进行动态调整，从而实现更灵活的布局和响应式设计。

**em（相对于父元素的字体大小）**：

- `em` 是相对于父元素的字体大小的单位。例如，如果一个元素的 `font-size` 设置为 `1.5em`，那么它的字体大小将会是父元素字体大小的 1.5 倍。
- `em` 单位的计算会受到父元素的字体大小影响，因此它适用于需要相对于父元素进行调整的元素尺寸。
- 缺点是，多层级的嵌套会导致字体大小递增，可能会产生意想不到的结果。

**rem（相对于根元素的字体大小）**：

- `rem` 是相对于根元素（通常是 `<html>` 元素）的字体大小的单位。它在整个页面中保持一致，不受父元素影响。
- `rem` 单位适用于全局样式，特别是在响应式设计中，可以通过设置根元素的字体大小来调整整个页面的比例。
- 由于 `rem` 不受父元素影响，避免了多层级嵌套导致的字体大小累加问题。

使用示例：

```css
/* 使用 em */
.parent {
  font-size: 16px;
}

.child {
  font-size: 1.5em; /* 等同于 24px（1.5 * 16px） */
}

/* 使用 rem */
html {
  font-size: 16px; /* 设置根元素字体大小 */
}

.parent {
  font-size: 1.5rem; /* 24px（1.5 * 16px） */
}

.child {
  font-size: 1rem; /* 16px（1 * 16px），与根元素字体大小相同 */
}
```

总的来说，`em` 和 `rem` 单位可以帮助实现更灵活的布局和响应式设计，使元素尺寸和字体大小能够根据不同层级或全局设置进行自适应调整。`rem` 尤其适合于在响应式设计中使用，可以有效地控制页面的比例。

### 各自相对优点

`em` 和 `rem` 都是相对单位，它们相对于绝对单位（如像素）具有一些优点，尤其在响应式设计和字体大小调整方面。

**`em` 相对于 `rem` 的优点**：

1. **相对于父元素的字体大小**：`em` 单位是相对于父元素的字体大小的，这使得它特别适合用于在组件内部实现相对尺寸。例如，你可以使用 `em` 来设置按钮、标题等组件的大小，以确保它们与周围文本的比例一致。

2. **多层级嵌套**：在多层级嵌套的情况下，`em` 可以很好地进行相对调整。每个嵌套级别都相对于其父元素的字体大小进行计算，可以实现逐级递增或递减的效果。

3. **局部控制**：使用 `em` 可以在组件级别对字体大小和元素尺寸进行微调，而不必改变全局设置。

**`rem` 相对于 `em` 的优点**：

1. **相对于根元素的字体大小**：`rem` 单位是相对于根元素的字体大小的，因此它在整个页面中保持一致。这可以使页面中的元素具有一致的比例。

2. **避免多层级影响**：由于 `em` 是相对于父元素的字体大小的，多层级嵌套可能导致字体大小累加的问题。`rem` 单位避免了这种问题，因为它不受父元素影响。

3. **响应式设计**：`rem` 特别适用于响应式设计，通过调整根元素的字体大小，可以同时调整整个页面的比例，从而实现更灵活的适应性。

4. **全局控制**：通过设置根元素的字体大小，你可以在整个页面范围内统一调整元素的尺寸和字体大小，而不必逐个元素进行调整。

综上所述，`em` 和 `rem` 都有各自的优势，适用于不同的情况。`em` 更适合于局部调整和相对尺寸，而 `rem` 更适合于全局调整和一致的比例。在实际应用中，你可以根据具体需求选择合适的相对单位。

## after是伪类选择器还是伪元素选择器

`::after` 是一个伪元素选择器，而不是伪类选择器。

伪元素是用来向文档中添加虚拟的元素，以便在 CSS 中为其应用样式。`::after` 伪元素用于在选定元素的内容之后插入一个虚拟元素，并且你可以为这个虚拟元素应用样式。

示例：

```css
p::after {
  content: " (Added content)";
  color: red;
}
```

在上面的示例中，`::after` 伪元素被用来在每个 `<p>` 元素的内容后面插入带有红色文本的附加内容。

与伪元素不同，伪类（如 `:hover`、`:active` 等）用于在特定的状态下选择元素，而不是在元素内部插入虚拟元素。

## 视口单位vw

`vw` 是 CSS3 中的一个相对单位，表示视口宽度的百分比。视口（viewport）是指浏览器中用于显示网页内容的可视区域。`1vw` 等于视口宽度的 1%。

使用 `vw` 单位可以使元素的尺寸与视口的大小成比例地变化，从而实现响应式设计。这对于创建适应不同屏幕宽度的布局和元素尺寸非常有用。

示例：

```css
.container {
  width: 80vw; /* 元素宽度为视口宽度的 80% */
  padding: 2vw; /* 内边距为视口宽度的 2% */
  font-size: 3vw; /* 字体大小为视口宽度的 3% */
}
```

在这个示例中，`.container` 元素的宽度、内边距和字体大小都使用了 `vw` 单位，这样它们将随着视口的宽度变化而自适应调整。

优点：

- 能够实现相对于视口宽度的自适应布局。
- 在响应式设计中，可以在不同屏幕尺寸上调整元素的大小，以保持合适的比例。
- 不受父元素的影响，适合于一些全局布局设置。

注意事项：

- 在某些情况下，`vw` 可能导致元素过大或过小，特别是在视口宽度极小或极大的情况下。需要合理地使用和测试。
- 在设置字体大小时，需要注意确保字体在小屏幕上不会变得太小，影响可读性。
- 不同浏览器和设备上的视口大小可能不同，所以在使用 `vw` 时要进行适当的测试和调整。

总的来说，`vw` 是一种有用的相对单位，适合于在响应式设计中实现视口宽度相关的自适应布局。

## [使用视口单位实现响应式排版](https://developer.mozilla.org/zh-CN/docs/Learn/CSS/CSS_layout/Responsive_Design#使用视口单位实现响应式排版)

> 如何理解这段话：问题在于，当做上面的事情的时候，因为文本总是随着视口的大小改变大小，用户失去了放缩任何使用vw单位的文本的能力。所以你永远都不要只用 viewport 单位设定文本。 这里有一个解决方法，它使用了calc()，如果你将vw单位加到了使用固定大小（例如em或者rem）的值组，那么文本仍然是可放缩的。基本来说，是vw加在了放缩后的值上。
>
> ---
>
> 这段话指的是使用 `vw` 单位设置文本大小可能会导致用户无法通过浏览器放缩来调整文本的大小。这是因为 `vw` 单位是相对于视口宽度的，如果所有文本都使用 `vw` 单位进行设置，那么当视口大小变化时，文本也会自动变化大小，而用户无法通过浏览器的放大缩小功能来调整文本大小。

> 为了解决这个问题，可以使用 `calc()` 函数，将 `vw` 单位与固定大小的单位（如 `em` 或 `rem`）结合使用。通过将 `vw` 单位应用于经过缩放的值，可以使文本仍然具有一定的可调性。基本上，你将 `vw` 单位与放缩后的单位组合在一起。
>
> 例如，如果你使用以下样式：
>
> ```css
> .text {
>   font-size: calc(1rem + 1vw);
> }
> ```
>
> 在这个样式中，文本的字体大小将基于基础的 `1rem` 大小，并且根据视口的宽度增加 `1vw`。这使得文本大小仍然可以通过浏览器的放大缩小功能进行调整，因为其中有一个基于 `rem` 的固定部分。
>
> 总之，这个建议是为了确保在使用视口单位设置文本大小时，用户仍然能够通过浏览器的文本缩放来进行调整，以提供更好的可用性和用户体验。

## Promise.prototype.then() 返回promise，this指针被包含在promise中吗

是的，`Promise.prototype.then()` 方法返回一个新的 Promise 对象，并且在新的 Promise 完成时会将前一个 Promise 的返回值传递给 `.then()` 中的回调函数。这意味着你可以在 `.then()` 中链式调用多个 `.then()` 方法，每个 `.then()` 返回的 Promise 都与前一个 `.then()` 的返回值相关联。

此外，`.then()` 中的回调函数内部的 `this` 指向取决于你如何定义这个回调函数。如果你使用了箭头函数，那么箭头函数内部的 `this` 将保留其外部上下文的值。如果你使用了普通函数，那么 `this` 将取决于函数被调用的方式。这与 Promise 本身没有直接关系，而是与 JavaScript 函数的工作原理相关。

示例：

```javascript
const promise = new Promise((resolve, reject) => {
  resolve(42);
});

promise.then(function(result) {
  console.log(result); // 输出 42
  console.log(this);    // 输出默认上下文（通常是 undefined 或全局对象）
});
```

```javascript
const promise = new Promise((resolve, reject) => {
  resolve(42);
});

promise.then((result) => {
  console.log(result); // 输出 42
  console.log(this);   // 输出包含 `.then()` 的外部上下文
});
```

需要注意的是，`this` 在 Promise 中的行为与 `.then()` 没有直接关系，而是由函数的定义方式和调用方式来决定。 Promise 自身并没有显式地将 `this` 绑定到 Promise 对象。

## Promise 的链式调用

```html
Promise.prototype.then()、Promise.prototype.catch() 和 Promise.prototype.finally() 方法用于将进一步的操作与已敲定的 Promise 相关联。由于这些方法返回 Promise，因此它们可以被链式调用。

.then() 方法最多接受两个参数；第一个参数是 Promise 兑现时的回调函数，第二个参数是 Promise 拒绝时的回调函数。每个 .then() 返回一个新生成的 Promise 对象，这个对象可被用于链式调用，例如：

const myPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("foo");
  }, 300);
});

myPromise
  .then(handleFulfilledA, handleRejectedA)
  .then(handleFulfilledB, handleRejectedB)
  .then(handleFulfilledC, handleRejectedC);
```

> ### resolve("foo"); 这个代码怎么理解
>
> `resolve("foo")` 是在 Promise 构造函数中使用的一个函数，用于将一个 Promise 对象从未完成状态（pending）变为已完成状态（fulfilled），并指定解决（fulfillment）后的值为 "foo"。
>
> 在 Promise 构造函数中，有两个参数：`resolve` 和 `reject`，它们都是函数。当 Promise 成功地完成时，你会调用 `resolve` 函数，并将最终的成功值作为参数传递进去。
>
> 在你的示例中：
>
> ```javascript
> const myPromise = new Promise((resolve, reject) => {
>   setTimeout(() => {
>     resolve("foo");
>   }, 300);
> });
> ```
>
> 在这里，`myPromise` 是一个 Promise 对象，它会在 300 毫秒后成功地解决为 "foo"。当 300 毫秒过去后，`resolve("foo")` 将被调用，将 Promise 对象从未完成状态变为已完成状态，并传递 "foo" 作为解决后的值。
>
> 总之，`resolve("foo")` 是 Promise 构造函数内部的一个函数调用，用于表示 Promise 成功完成，并将成功的值设置为 "foo"。
>
> ### 如果第二个.then是reject，会执行第三个.then吗
>
> 如果在 Promise 链中的任何一个 `.then()` 方法中出现拒绝（`reject`），那么控制流将跳过后续的 `.then()` 方法，直接进入最近的错误处理部分，也就是 `.catch()` 或后续的 `.then()` 方法中的错误处理回调函数。
>
> 所以，如果第二个 `.then()` 中出现了拒绝，第三个 `.then()` 将不会执行。相反，控制流将直接跳到最近的错误处理部分。这个行为是 Promise 链式调用的特性，它确保了在链式调用中的任何一个环节出现错误时，后续环节都会被跳过，以避免处理无效或错误的数据。
>
> 示例：
>
> ```javascript
> const myPromise = new Promise((resolve, reject) => {
>   setTimeout(() => {
>     resolve("foo");
>   }, 300);
> });
> 
> myPromise
>   .then((result) => {
>     console.log(result); // 输出 "foo"
>     throw new Error("Some error occurred");
>   })
>   .then(() => {
>     console.log("This won't be executed");
>   })
>   .catch((error) => {
>     console.error("Error caught:", error); // 输出 "Error caught: Some error occurred"
>   });
> ```
>
> 在这个示例中，第一个 `.then()` 方法中使用 `throw new Error()` 触发了拒绝。这导致后续的 `.then()` 方法被跳过，而是进入 `.catch()` 部分，捕获了错误。第二个 `.then()` 中的代码将不会执行。这展示了在 Promise 链中拒绝后如何处理错误。
>
> ### 如果.catch后面还有.then会正常执行是吧
>
> 是的，`.catch()` 方法返回的是一个 Promise 对象，因此你可以在 `.catch()` 后面继续使用 `.then()` 方法来继续处理。`.catch()` 可以用来捕获前面任何一个 Promise 链中出现的拒绝，然后你可以在 `.catch()` 后面对错误进行处理，之后继续链式调用。
>
> 示例：
>
> ```javascript
> const myPromise = new Promise((resolve, reject) => {
>   setTimeout(() => {
>     reject(new Error("Some error occurred"));
>   }, 300);
> });
> 
> myPromise
>   .then((result) => {
>     console.log(result); // 不会执行
>   })
>   .catch((error) => {
>     console.error("Error caught:", error); // 输出 "Error caught: Some error occurred"
>     return "Recovered from error";
>   })
>   .then((recoveryResult) => {
>     console.log("Recovery:", recoveryResult); // 输出 "Recovery: Recovered from error"
>   });
> ```
>
> 在这个示例中，`myPromise` 在 300 毫秒后拒绝，并抛出一个错误。第一个 `.then()` 不会执行，而是跳到 `.catch()` 部分，捕获了错误，并返回一个恢复的结果 "Recovered from error"。然后，第二个 `.then()` 接收到 "Recovered from error" 并正常执行。这展示了在 `.catch()` 后面继续使用 `.then()` 来处理错误和后续的操作。
